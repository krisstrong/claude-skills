# File Naming — Final Deliverables

Use this skill whenever the user asks to name, save, rename, or finalize a deliverable file.

## Naming Convention

All final deliverable files must follow this format:

```
DEPARTMENT OR COMPANY - SHORT NAME OF FILE - YYYYMMDD
```

**Examples:**
- `Marketing - Q2 Campaign Brief - 20260628`
- `Acme Corp - Executive Summary - 20260628`
- `Finance - Budget Forecast FY27 - 20260628`

## Rules

1. **Department or Company** — Use the client name, internal department, or team the file belongs to. Title case. No abbreviations unless widely understood (e.g., HR, IT).
2. **Short Name of File** — A concise, descriptive label for the document. Title case. 3–6 words max. No filler words (e.g., avoid "Final", "Draft", "v2" — those belong in version control, not the filename).
3. **Date** — Always use today's date in `YYYYMMDD` format. Never use relative dates ("today", "this week").
4. **Separator** — Use ` - ` (space-hyphen-space) between each segment. No underscores, slashes, or other separators.
5. **File extension** — Preserve the original file extension (e.g., `.docx`, `.pdf`, `.xlsx`).

## How to Apply

When the user asks to name or finalize a file:

1. Identify the three components from context. If any are ambiguous or missing, ask:
   - "Who is this for — which department or company?"
   - "What's a short name for this document?"
2. Get today's date and format it as `YYYYMMDD`.
3. Assemble the filename and confirm it with the user before applying.
4. If renaming an existing file, show the before and after:
   - **Before:** `old_filename.pdf`
   - **After:** `Marketing - Campaign Brief - 20260628.pdf`
